There is a set of ideas that is traditionally considered the hallmark of OOP - abstraction, encapsulation, inheritance and polymorphism. Clojure, while being a functional programming language, has adapted these ideas to fit the functional programming paradigm, in a way that merges the best of two worlds. In this talk I will present this adaptation, and show how it still possible to use the powerful aspects of OOP, while enjoying doing FP, Clojure style.<br>
