Let's discuss some "provocative" ideas in Software Development, that beyond being provocative - make some sense:<br>
Inheritance is a bold violation of OOP Principles.<br>
Duplicate code when you are not sure about the abstraction.<br>
Copy-paste as a legit method of code reuse.<br>
Git is not so good for you.<br>
You spend too much on Unit Tests.<br>
Reduce testing, and replace it with better production monitoring.</div>
