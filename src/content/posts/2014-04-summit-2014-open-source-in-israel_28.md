<div style="background-color: rgba(255, 255, 255, 0.498039); box-sizing: border-box; color: #333333; font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif; font-size: 14px; line-height: 20px; margin-bottom: 10px; word-break: break-word;">
Today's Devops ecosystem consists of diverse set of tools, configuration management (Chef/Puppet), remote execution (Fabric, Capistrano), monitoring and central logging all used against a large number of systems (optionally) spread on multiple hypervisors (AWS, vCenter, Docker, Proxmox etc..),</div>
<div style="background-color: rgba(255, 255, 255, 0.498039); box-sizing: border-box; color: #333333; font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif; font-size: 14px; line-height: 20px; margin-bottom: 10px; word-break: break-word;">
All these tools work great but don't give us a cohesive system to work with, we still need to:</div>
<ul style="background-color: rgba(255, 255, 255, 0.498039); box-sizing: border-box; color: #333333; font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif; font-size: 14px; line-height: 20px; margin-bottom: 10px; margin-top: 0px;">
<li style="box-sizing: border-box;">Have UI and API</li>
<li style="box-sizing: border-box;">Allow users to self service (monitor resource usage, manage it etc..)</li>
<li style="box-sizing: border-box;">Support multiple platforms transparently (AWS, vCenter, Docker etc..) without having to reinvent the wheel each time.</li>
<li style="box-sizing: border-box;">Have a process not just another tool, how other parties add content to configuration management, how we deploy the changes etc..</li>
<li style="box-sizing: border-box;">Manage resources intelligently (between users)</li>
<li style="box-sizing: border-box;">Minimize manual book keeping (assigning static ips, register dns records etc..)</li>
</ul>
<div style="background-color: rgba(255, 255, 255, 0.498039); box-sizing: border-box; color: #333333; font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif; font-size: 14px; line-height: 20px; margin-bottom: 10px; word-break: break-word;">
Celestial comes to fill this void, it uses well known best practices from actual field work:</div>
<ul style="background-color: rgba(255, 255, 255, 0.498039); box-sizing: border-box; color: #333333; font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif; font-size: 14px; line-height: 20px; margin-bottom: 10px; margin-top: 0px;">
<li style="box-sizing: border-box;">End to end life cycle management of systems with simple understandable model.</li>
<li style="box-sizing: border-box;">An easy to use UI and Restful API, Hubot integration.</li>
<li style="box-sizing: border-box;">User, Quota and multiple environments management.</li>
<li style="box-sizing: border-box;">Distributed locking, complete auditing of all actions with easy to slice through structure in kibana/graylog2</li>
<li style="box-sizing: border-box;">Integration with leading Devops tools.</li>
<li style="box-sizing: border-box;">Hand crafted fine tuned support for leading hypervisors.</li>
</ul>
<div style="background-color: rgba(255, 255, 255, 0.498039); box-sizing: border-box; color: #333333; font-family: 'Helvetica Neue', Helvetica, Arial, sans-serif; font-size: 14px; line-height: 20px; margin-bottom: 10px; word-break: break-word;">
In this talk we will cover Celestial and see it action.</div>
