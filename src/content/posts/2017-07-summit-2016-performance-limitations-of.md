How well can React Native hold its own against purely native implementations?</span><br style="background-color: white; color: #333333; font-family: &quot;YouTube Noto&quot;, Roboto, arial, sans-serif; font-size: 13px;"><br style="background-color: white; color: #333333; font-family: &quot;YouTube Noto&quot;, Roboto, arial, sans-serif; font-size: 13px;"><span style="background-color: white; color: #333333; font-family: &quot;YouTube Noto&quot;, Roboto, arial, sans-serif; font-size: 13px;">Background</span><br style="background-color: white; color: #333333; font-family: &quot;YouTube Noto&quot;, Roboto, arial, sans-serif; font-size: 13px;"><span style="background-color: white; color: #333333; font-family: &quot;YouTube Noto&quot;, Roboto, arial, sans-serif; font-size: 13px;">I’ve been developing native mobile apps professionally for the past 7 years. I’ve seen many magical “cross-platform” solutions but eventually nothing beats pure-native in terms of app quality. The main thing developers are skeptical of is performance. React Native is unique in its ability to achieve amazing performance, but it requires developers to understand what goes on under the hood. The aim of this talk is to ease concerns for new-comers and teach existing developers a few important guidelines.</span></div>
